Spatial Joins

A spatial join is a technique used in geographic information systems (GIS) to combine data from two datasets based on their spatial relationship. It is similar to a database join but is based on the geometry of spatial features, such as points, lines, or polygons, instead of attribute values.

Key Features of Spatial Joins

  1. Spatial Relationships:

    • Proximity: Joining features based on whether one is near another.

    • Containment: Joining features that are completely within another feature.

  2. Input Data:

    • Usually involves two spatial datasets:

      • Target dataset: The main dataset where attributes are appended.

      • Join dataset: The dataset providing the additional attributes.

  3. Result:

    • A new dataset is created, combining attributes from both datasets based on their spatial relationship.

Common Examples

  • Assigning Points to Polygons: Assigning city points to the state polygons they belong to.

  • Proximity Analysis: Finding the nearest hospital for each school.

Count Points in a Polygon

Often we want to know how many points fall within a polygon. This can be helpful to calculate rates or prevalence by a given polygon. An example may be the number of tuberculosis cases in a given census tract.

For this example we want to calculate the number of cholera deaths per Soho neighborhood block in our data during period two of the cholera outbreak.

This will require the target dataset, neighborhoods, and the join dataset deaths.

The function we will use is st_join and it takes three arguments:

  • the points data

  • the polygon data

  • and the join type, in this case st_within to count the points that fall within a polygon

We will also use st_drop_geometry so that we can remove the geometry to aggregate the cases within the neighborhoods.

This process will take a few steps, let’s look at them

  1. Step One: We get a joined dataset that has repeated polygons, one for each death. This will not map, we need to aggregate to count those cases. This only links the cases to the polygon.

    points_in_polygons <- st_join(deaths, neighborhoods, join=st_within)

  2. Step Two: We must remove the geometry using st_drop_geometry so that we can aggregate and count the number of cases per polygon. We will group_by and the Id variable to get the counts. Notice, if we look at the number of rows in the point_counts dataset there are only 51, that means only 51 of the polygons have cases.

    point_counts <- points_in_polygons %>%
  st_drop_geometry() %>%
  count(Id)

  3. Step Three: We now have the point_counts dataset which has the counts by polygon Id lets table join to the neighborhoods by Id to get the count data in the neighborhoods.

    neighborhoods_death <- neighborhoods |> 
                       inner_join(point_counts, by=join_by("Id"))

  4. Step Four: Let’s map this thing!

    tm_shape(neighborhoods)+
  tm_polygons()+
  tm_shape(neighborhoods_death)+
  tm_fill(col = "n", title="Cholera Cases" )

Find the Closest Point

Now, what if we want to know which pump is closest to each case? That is a spatial join as well.

Lets use a function called st_nearest_feature which takes two arguments:

  • x for each item in this object we will find its closest feature in y

  • y our object that holds the features were searching for

    nearest <- deaths |>

Kernel Density Estimate

We can now create a heatmap of the cases so that we can where the cases happen most often.

  1. Step One: Define the Study Area by defining a bounding polygon or a grid for KDE calculation. cell_size here is the size of the grid cells we wish to use based on the units of the features. In this case deaths is in British National Grid projection so it is in meters. So cell_size 1 is one meter by one meter. This may take a bit!

    # Create a study grid over the area
grid <- create_raster(deaths, cell_size = 1)

  2. Step Two: Calculate the kernel density, meaning how many cases happen in each of the cells of the raster grid. Wider bandwidth means ‘smoother’ rasters. This may take a bit!

    # KDE computation
kde_result_15 <- kde(
  points = deaths,
  band_width = 15,  # Adjust bandwidth as needed
  grid = grid
)
